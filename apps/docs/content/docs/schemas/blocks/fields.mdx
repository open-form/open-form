title: Fields
description: All field types for capturing data
---

Fields define the data structure of forms. Each field type captures specific kinds of data.

## Field Type Summary

| Type | Value Type | Key Properties |
|------|-----------|----------------|
| `text` | `string` | minLength, maxLength, pattern |
| `number` | `number` | min, max |
| `boolean` | `boolean` | default |
| `enum` | `string \| number` | enum (array of options) |
| `multiselect` | `(string \| number)[]` | options, min, max |
| `email` | `string` | minLength, maxLength |
| `uri` | `string` | minLength, maxLength, pattern |
| `uuid` | `string` | minLength, maxLength, pattern |
| `date` | `string` | min, max (ISO 8601 YYYY-MM-DD) |
| `datetime` | `string` | min, max (ISO 8601) |
| `time` | `string` | min, max (HH:MM:SS) |
| `duration` | `Duration` | default |
| `money` | `Money` | min, max |
| `percentage` | `number` | min, max, precision |
| `rating` | `number` | min, max, step |
| `address` | `Address` | default |
| `phone` | `Phone` | default |
| `coordinate` | `Coordinate` | default |
| `bbox` | `Bbox` | default |
| `person` | `Person` | default |
| `organization` | `Organization` | default |
| `identification` | `Identification` | allowedTypes |
| `fieldset` | `object` | fields (nested) |

## Common Properties

All field types share these base properties:

| Property | Type | Required | Description |
|----------|------|----------|-------------|
| `type` | `string` | Yes | Field type discriminator |
| `label` | `string` | No | Human-readable label for the field |
| `required` | `boolean \| string` | No | Whether this field is required. Can be a boolean or logic expression |
| `visible` | `boolean \| string` | No | Whether this field is visible. Can be a boolean or logic expression. Defaults to true |
| `description` | `string` | No | Long-form description or helper text displayed in the UI |
| `disabled` | `boolean \| string` | No | Whether the field is read-only/disabled. Can be a boolean or logic expression |
| `attrs` | `Record<string, string \| number \| boolean>` | No | Custom attribute-value pairs for field metadata |

## Text Field

Text input field.

| Property | Type | Required | Description |
|----------|------|----------|-------------|
| `type` | `'text'` | Yes | Field type discriminator |
| `minLength` | `number` | No | Minimum character length |
| `maxLength` | `number` | No | Maximum character length |
| `pattern` | `string` | No | Regex pattern for validation |
| `default` | `string` | No | Default value |

```yaml tab="YAML"
name:
  type: text
  label: Full Name
  required: true
  minLength: 2
  maxLength: 100
```

```json tab="JSON"
{
  "name": {
    "type": "text",
    "label": "Full Name",
    "required": true,
    "minLength": 2,
    "maxLength": 100
  }
}
```

## Fieldset Field

Nested field container for grouping related fields.

| Property | Type | Required | Description |
|----------|------|----------|-------------|
| `type` | `'fieldset'` | Yes | Literal 'fieldset' discriminator |
| `fields` | `Record<string, Field>` | Yes | Nested field definitions keyed by identifier |

```yaml tab="YAML"
contact:
  type: fieldset
  label: Contact Information
  fields:
    name:
      type: text
      label: Name
      required: true
    email:
      type: email
      label: Email
      required: true
    phone:
      type: phone
      label: Phone
```

```json tab="JSON"
{
  "contact": {
    "type": "fieldset",
    "label": "Contact Information",
    "fields": {
      "name": {
        "type": "text",
        "label": "Name",
        "required": true
      },
      "email": {
        "type": "email",
        "label": "Email",
        "required": true
      },
      "phone": {
        "type": "phone",
        "label": "Phone"
      }
    }
  }
}
```
