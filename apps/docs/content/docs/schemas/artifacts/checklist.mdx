---
title: Checklist
description: An artifact for tracking tasks and progress
---

A Checklist defines items to track within a workflow.

Each item can have a boolean (done/not done) or enumerated status (e.g., pending, approved, rejected).

Examples include a loan closing checklist, document review checklist, or onboarding checklist.

# Examples

```yaml tab="YAML"
kind: checklist
name: loan-closing
version: "1.0.0"
title: Loan Closing Checklist

items:
  - id: title-search
    title: Title Search
    description: Complete title search and review
    status:
      kind: boolean
      default: false

  - id: appraisal
    title: Appraisal
    status:
      kind: enum
      options:
        - value: pending
          label: Pending
        - value: approved
          label: Approved
        - value: rejected
          label: Rejected
      default: pending
```
